## @oracle second No-Go repair (binding)

After the first No-Go repair (which required FD1 private replay to
compute a real file-selector lower bound), @oracle flagged two remaining
blockers:

1. **FD1 private replay not fail-closed enough.** The workflow verified
   only that the private JSONL existed and had 86040 rows. This was
   insufficient: a stale / mismatched / partial replay could still
   authorize v1-A. The repaired workflow now validates the FD1 replay
   report (`fd1_replay_report.json`) before trusting the JSONL:
   schema_version, status, records_decomposed,
   private_decomposition_manifest (schema/record_count/records_written/
   path_publicly_serialized), manifest_hash match vs the committed FD1
   artifact, and forbidden_scan.status. The audit evaluator re-validates
   the replay report via `--fd1-replay-artifact` and No-Gos to
   `no_go_ceiling_unavailable` if the replay artifact is missing /
   invalid / mismatched (even when the private JSONL is supplied).

2. **Stale/ambiguous no-replay claims.** The previous
   `bea_v1_p1_no_replay_executed=true` flag was misleading because the
   CI workflow regenerates the FD1 private decomposition. Renamed to
   `bea_v1_p1_audit_evaluator_no_replay_executed=true` (the v1 audit
   *evaluator* itself does not run retrieval/selector/replay). Added
   explicit provenance flags:
   `fd1_private_decomposition_replay_supplied`,
   `fd1_private_decomposition_replay_validated`,
   `fd1_private_decomposition_replay_executed_by_workflow`. For the
   local default / no-private artifact these are all false (supplied
   false). For the real workflow report, supplied/validated true and
   workflow-executed true (when a validated replay artifact is present).

The audit evaluator does not rerun FD1; the manual CI workflow
regenerates the FD1 private decomposition under temp storage to compute
the lower bound. The public artifact remains aggregate-only and
records-only — no private JSONL paths or content hashes are serialized.
The `replay_artifact_manifest_hash` is the FD1 private decomposition
*schema manifest hash* (a schema-content hash), NOT a private JSONL
content hash, and must match the committed FD1 artifact's manifest hash.

## Action layers (6)

Each FD1 failure category is mapped to one of six BEA v1 action layers
that can causally affect it:

1. `candidate_availability_retrieval`
2. `file_selector`
3. `span_refiner`
4. `setwise_packer_redundancy`
5. `stopping_scheduler`
6. `non_actionable_accounting`

## Cell classes (5)

Each `(failure_category, action_layer)` cell is exactly one of:

- `direct_actionable` — layer is the primary fix point for this category
- `indirect_actionable` — layer can partially address the category
- `not_actionable_by_layer` — layer cannot causally affect the category
- `candidate_unavailable` — FD1 marks the category
  `unavailable_no_support_label` (no support/target labels in the
  private SCORE schema); candidate-level actionability cannot be
  evaluated; ceiling cannot be computed
- `ceiling_unavailable_insufficient_trace` — FD1 marks the category
  `unavailable_missing_trace` (per-record trace insufficient in the
  public aggregate); ceiling cannot be computed

## Actionability matrix (12 × 6 = 72 cells)

Public table `actionability_matrix_records` with natural key
`(failure_category, action_layer)` publishes all 72 cells. Highlights
(among the 7 available FD1 categories):

- `gold_file_absent` → `file_selector` is `direct_actionable`
  (a perfect selector that picks gold when it is in the candidate pool
  could recover this); `candidate_availability_retrieval` is
  `candidate_unavailable` (gold absent means retrieval layer cannot
  recover).
- `correct_file_wrong_span` and `gold_span_absent` → `span_refiner`
  is `direct_actionable`.
- `too_many_anchor_slots` → `span_refiner` is `direct_actionable`
  (anchor selection lives in the span layer).
- `early_stop_too_early` → `stopping_scheduler` is `direct_actionable`.
- `budget_spent_on_low_marginal_gain` → `setwise_packer_redundancy`
  is `direct_actionable` (marginal-utility packing controls budget).
- `latency_without_quality_gain` → `non_actionable_accounting` is
  `direct_actionable` (latency category is not actionable by
  candidate-level proxy during selection — it must be accounted for
  separately, not folded into selection loss). This is the FD2-A1
  No-Go bucket.

The 3 `unavailable_no_support_label` categories
(`missing_support_candidate`, `support_selected_without_target`,
`target_selected_without_support`) are uniform
`candidate_unavailable` across all 6 layers. The 2
`unavailable_missing_trace` categories
(`redundant_same_file_candidates`, `risk_penalty_removed_gold`) are
uniform `ceiling_unavailable_insufficient_trace` across all 6 layers.

## Oracle ceilings (honest, never inferred from aggregate latency)

### file_selector ceiling (REQUIRED, requires FD1 private replay)

Computed as **both upper and lower bound** from the FD1 public aggregate
+ the regenerated FD1 private decomposition JSONL:

- denominator = `gold_file_absent` affected record count across all
  source_phase/benchmark buckets = 119 (BEA-4 62 + BEA-4 repoqa 17 +
  BEA-5 24 + BEA-5 repoqa 16).
- recoverable upper bound = denominator = 119 (a perfect selector that
  picks gold whenever it is in the candidate pool could recover at
  most this many records).
- recoverable rate upper bound = 119 / 239 = 0.498.
- recoverable lower bound = count of denominator records where ANY
  baseline arm (v0.2 / v0 / bm25_prefix / agreement_only / rrf) has
  `baseline_value` for `file_recall@10` > 0.0 (another same-pool /
  same-frame arm selected a correct file → the gold was in the
  candidate pool). This is a LOWER BOUND on recoverability — it does
  NOT prove that every denominator record's gold was in the pool.
- unrecoverable_candidate_unavailable_count = denominator − lower
  bound (records whose gold was not reachable by any baseline arm on
  the same pool/frame).
- retrieval_availability_rate = unrecoverable / denominator (if
  denominator > 0).
- ceiling_class = `computed_private_lower_bound_and_public_upper_bound`;
- ceiling_basis = `fd1_private_decomposition_replay`.

If the private JSONL is not supplied (or unparseable, or its row count
≠ 86040, or its grouped record count ≠ 239), the file_selector row is
omitted from `oracle_ceiling_records`, an explicit
`unavailable` row is published in `unavailable_ceiling_records`, and
the audit No-Gos to `no_go_ceiling_unavailable` with
`stop_go_decision = needs_fd1_private_replay_before_v1_a`.

### span_refiner ceiling (unavailable)

FD1 has aggregate `correct_file_wrong_span` counts but no per-record
candidate/gold span overlap. Public aggregate is insufficient to bound
the span-refiner ceiling. Published as an explicit
`unavailable_ceiling_records` row with reason
`candidate_or_gold_span_overlap_fields_absent_in_fd1_public_aggregate`.

### setwise_packer_redundancy ceiling (unavailable)

FD1 marks `redundant_same_file_candidates` as
`unavailable_missing_trace` (the duplicate / file-grouping fields are
not in the public aggregate). Published as an explicit
`unavailable_ceiling_records` row with reason
`redundant_same_file_candidates_marked_unavailable_missing_trace_in_fd1`.

### stopping_scheduler ceiling (unavailable)

FD1 has aggregate `latency_loss` per category but no ordered-prefix
utility/latency. The plan forbids inferring stopping ceiling from
aggregate latency loss. Published as an explicit
`unavailable_ceiling_records` row with reason
`ordered_prefix_quality_or_latency_absent_in_fd1_public_aggregate`.

## Stop / go rule for v1-A coverage-preserving selector

Go to BEA-v1-A (a runtime-clean coverage-preserving file selector)
only if ALL of the following hold:

1. FD1 private decomposition replay was parsed (86040 rows, 239 grouped
   records, zero parse failures). Audit requires this — without it the
   audit No-Gos to `no_go_ceiling_unavailable` and stop_go is
   `needs_fd1_private_replay_before_v1_a`.
2. File-selector oracle ceiling computed with nonzero denominator AND
   a non-null lower bound (`file_selector_ceiling_computed == true`).
3. File-selector lower bound rate is material
   (`file_selector_lower_bound_rate >= 0.05`). The lower bound rate
   (NOT the upper bound rate) drives the materiality test.
4. Retrieval availability does not dominate
   (`retrieval_availability_rate <= 0.50` of denominator records).
5. Span / stopping issues do not dominate the recoverable ceiling
   (`span_or_stopping_dominates == false`, computed as
   span_or_stopping dominance rate < file_selector lower bound rate
   AND < 0.50 threshold).
6. A runtime-clean coverage-preserving selector is plausible
   (BEA-v1-P1 does not execute any selector; this is a structural
   judgment based on the actionability matrix showing
   `file_selector` is the direct_actionable layer for
   `gold_file_absent` and the FD1 frame is frozen).

Audit `stop_go_decision = go_v1_a_coverage_preserving_selector` only
when all conditions hold.

If any condition fails, BEA-v1-P1 stops and recommends either
rerunning the FD1 private decomposition replay (if condition 1 fails)
or moving to the layer with the highest honest ceiling (if a later
condition fails).

If any condition fails, BEA-v1-P1 stops and recommends moving to the
layer with the highest honest ceiling.

## Public artifact contract

Aggregate-only, records-only. No public record IDs, paths, queries,
snippets, spans, candidate keys, selected order, private trace paths,
or private row payloads. Only counts, rates, hashes, schema names, and
aggregate metrics.

Required public tables (records-only, natural keys):

- `source_run_records`: `(source_phase, source_ci_run_id)` — the FD1
  audit source (committed FD1 artifact) with verification fields
  (expected vs audited counts, FD1 schema/hash/status, FD2-A1
  binding-context schema/hash/status/checkpoint), AND the FD1 private
  decomposition replay status
  (`fd1_private_decomposition_supplied`, `fd1_private_decomposition_parsed`,
  `fd1_private_decomposition_row_count` must be 86040,
  `fd1_private_decomposition_group_count` must be 239,
  `fd1_private_decomposition_denominator`,
  `fd1_private_decomposition_lower_bound`).
- `failure_category_records`: `(failure_category,)` — 12 rows; per
  category affected count (summed across source_phase/benchmark
  buckets) and FD1 availability class.
- `actionability_matrix_records`: `(failure_category, action_layer)`
  — 72 rows (12 categories × 6 layers); cell class plus boolean
  is_direct_actionable / is_indirect_actionable /
  is_candidate_unavailable / is_ceiling_unavailable flags.
- `oracle_ceiling_records`: `(ceiling_name,)` — computed ceilings
  (file_selector only). `oracle_ceiling_records` is reserved for
  ceilings that were actually computed; unavailable ceilings are
  published in `unavailable_ceiling_records` instead.
- `candidate_availability_records`:
  `(source_phase, benchmark, failure_category)` — per-(sp, bm, cat)

  (span_refiner, setwise_packer_redundancy, stopping_scheduler)

- `redundancy_tradeoff_records`: `(tradeoff_axis,)` — 2 rows
  (marginal_utility_per_added_candidate,
  duplicate_same_file_suppression_cost), both `unavailable` with
  reason.
- `stop_go_records`: `(stop_go_decision,)` — 1 row; v1-A stop/go
  decision and reason, with all threshold inputs recorded.
- `gate_records`: `(gate,)` — 11 fail-closed gates.
- `private_manifest_records`: `(manifest_name,)` — 1 row echoing the

- `failure_category_count_records`: `(failure_category,)` — audit-level

## Caveats

- BEA-v1-P1 is eval/diagnostic only. NOT benchmark/leaderboard/
  performance/method-winner/calibration/promotion/default/runtime/
  EvidenceCore/downstream-value claim.
- The file-selector oracle ceiling requires the FD1 private
  decomposition replay (86040 rows / 239 grouped records) AND the FD1
  replay report (validated: schema/status/counts/manifest-hash/
  forbidden_scan) to compute a real lower bound. The public aggregate
  upper bound alone is explicitly insufficient to authorize v1-A —
  without the private replay AND the validated replay artifact, the
  audit honestly No-Gos to `no_go_ceiling_unavailable`.
- The audit *evaluator* does not rerun FD1, does not run
  retrieval/selector/provider calls. The manual CI workflow
  regenerates the FD1 private decomposition under `/tmp` (deterministic
  frozen BEA-4/5 replay with no provider calls) and writes the FD1
  replay report; the audit evaluator reads both. The
  `bea_v1_p1_audit_evaluator_no_replay_executed=true` flag tracks the
  evaluator's no-replay invariant; the
  `fd1_private_decomposition_replay_*` flags track the workflow's
  replay provenance.
- The recoverable lower bound is a *lower bound only*: it counts
  records where ANY baseline arm selected a correct file. It does NOT
  prove full candidate-pool availability for every denominator record.
- The `replay_artifact_manifest_hash` is the FD1 private decomposition
  *schema manifest hash* (a schema-content hash), NOT a private JSONL
  content hash. No private JSONL paths or content hashes are
  serialized in the public artifact.
- The span-refiner / setwise_packer_redundancy / stopping_scheduler
  ceilings are honestly unavailable. The plan forbids inferring the
  stopping ceiling from aggregate latency loss.
- The audit inherits FD1's unavailable-category discipline:
  `redundant_same_file_candidates` and `risk_penalty_removed_gold`
  are `unavailable_missing_trace`; `missing_support_candidate`,
  `support_selected_without_target`, and
  `target_selected_without_support` are `unavailable_no_support_label`.
  BEA-v1-P1 does NOT invent support/target labels or per-record
  traces.
- BEA-v1-P1 is NOT BEA v0.4 repair, NOT FD2-B, NOT FD2-C, NOT P4,
  NOT P5, NOT v0.31/v0.32 tuning, NOT B16-K, NOT a re-run of FD2-A1.
  The framing flags `is_v04_repair=false`, `is_fd2_b=false`,
  `is_fd2_c=false`, `is_p4=false`, `is_p5=false`,
  `is_v031_tuning=false`, `is_v032_tuning=false`, `is_b16k=false`
  are binding.
